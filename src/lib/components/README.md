# `lib/components/`

Svelte components, organized by purpose.

## Folders

- `ui/` — **shadcn-svelte primitives** (Button, Dialog, Input, etc.). Generated by the shadcn CLI. **Do not edit by hand** — re-generate or override via wrapper components in `shared/` instead.
- `viewer/` — Feature components for the viewer route: toolbars, sidebars, summary tab, citation pills, popovers, export controls.
- `dashboard/` — Feature components for the dashboard: search bar, document cards, Deep mode entry.
- `shared/` — Cross-route components and app-shell pieces (sidebar, user menu, theme toggle, etc.).

## Conventions

- Feature components import primitives from `ui/` rather than from `bits-ui` directly — keeps the shadcn override path consistent.
- One component per file. PascalCase filenames (e.g. `CitationPill.svelte`).
- Component-specific state lives in `.svelte.ts` siblings (Svelte 5 runes).
